返回码（respCode） 返回码信息（respMsg） 备注
0000 下单成功 该返回码表示下单成功。
0001 充值中 该返回码表示正在充值中。
0002 充值成功 该返回码表示充值成功。
0003 充值失败 该返回码表示充值失败。
1000 用户不存在 请核对账号名称(account)是否正确。
1001 IP鉴权失败 鉴权失败，请核对你的服务器请求IP与所填写的IP鉴权地址。备注：空或者255.255.255.255表示不鉴权。
1002 签名校验失败 .签名操作必须完全按照接口文档要求拼接后，将用户获取的apikey 拼接在参数的首尾进行md5加密。
1003 该订单不存在 请核对订单号，确认订单是否存在。
1004 您无权查看别人的订单状态 核对所请求的订单是否为当前账号。
1005 参数提交不完全 请核对你所提交的参数。
3002 无效手机号 请核对你提交的手机号码。
3003 无效区域代号 请核对手机号的和充值的地区标识。
3004 无效流量包大小 请核对提交的订单流量包size值。
3005 无法找到相应的产品 请核对是否存在相对于的产品
3006 无法找到相应的库存 请核对你的库存。
3007 用户无权限给此运营商的手机号充流量 请核对是的运营商权限。
4000 余额不足 请核对你的账户余额。
4001 余额不足（上级） 请联系上级，咨询余额是否不足。
4002 缓存订单余额不足 用户缓存订单余额不足。
4003 号码充值失败次数过多 号码充值失败次数过多, 请暂时不要再试。
8000 其他错误。请联系工程师跟进。 请联系技术。
备注:
1.开发者接口开发者，反馈错误信息请先核对上面返回码进行排查。
2.如还未及时解决请联系相关技术人员。
